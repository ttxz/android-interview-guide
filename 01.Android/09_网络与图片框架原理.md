# Android 网络与图片框架原理

## 一、OkHttp 原理

> **面试核心考点**：拦截器链、连接池、缓存

### 1. 拦截器链（责任链模式）

```
请求发出 →
  RetryAndFollowUpInterceptor    → 重试与重定向
  BridgeInterceptor              → 补充请求头（Content-Type、Cookie、gzip）
  CacheInterceptor               → 缓存判断（304）
  ConnectInterceptor             → 建立连接（TCP + TLS）
  CallServerInterceptor          → 发送请求、读取响应
← 响应返回
```

```kotlin
// 自定义拦截器
class LoggingInterceptor : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val request = chain.request()
        val startTime = System.nanoTime()

        Log.d("HTTP", "→ ${request.method} ${request.url}")

        val response = chain.proceed(request)  // 继续链条

        val duration = (System.nanoTime() - startTime) / 1e6
        Log.d("HTTP", "← ${response.code} (${duration}ms)")

        return response
    }
}

// 应用拦截器 vs 网络拦截器
val client = OkHttpClient.Builder()
    .addInterceptor(LoggingInterceptor())        // 应用拦截器：最先执行
    .addNetworkInterceptor(CacheInterceptor())   // 网络拦截器：在连接后执行
    .build()
```

| 对比 | 应用拦截器 | 网络拦截器 |
|------|----------|----------|
| 位置 | 最外层 | ConnectInterceptor 之后 |
| 重定向 | 只调用一次 | 每次网络请求都调用 |
| 缓存响应 | 不会触发 | 可以观察缓存响应 |
| 适用场景 | 日志、统一头部 | 缓存控制、压缩 |

### 2. 连接池（ConnectionPool）

```kotlin
// 默认：最多 5 个空闲连接，每个存活 5 分钟
val pool = ConnectionPool(
    maxIdleConnections = 5,
    keepAliveDuration = 5,
    timeUnit = TimeUnit.MINUTES
)

// 连接复用条件：
// 1. 相同的 host + port
// 2. 相同的 TLS 设置
// 3. HTTP/2 支持多路复用（一个连接并发多个请求）
```

### 3. 缓存策略

```kotlin
// 设置缓存
val cache = Cache(cacheDir, 10 * 1024 * 1024) // 10MB
val client = OkHttpClient.Builder().cache(cache).build()

// CacheControl
val request = Request.Builder()
    .url(url)
    .cacheControl(CacheControl.Builder()
        .maxAge(5, TimeUnit.MINUTES)   // 5分钟内使用缓存
        .build())
    .build()

// 强制缓存 / 强制网络
CacheControl.FORCE_CACHE    // 仅读缓存
CacheControl.FORCE_NETWORK  // 仅走网络
```

## 二、Retrofit 原理

> **面试常问**：Retrofit 的核心设计

### 1. 动态代理

```kotlin
// 定义接口
interface ApiService {
    @GET("users/{id}")
    suspend fun getUser(@Path("id") id: Long): User

    @POST("users")
    suspend fun createUser(@Body user: User): User

    @GET("users")
    suspend fun getUsers(@Query("page") page: Int): List<User>
}

// Retrofit.create() 内部使用动态代理
val service = retrofit.create(ApiService::class.java)
// 等价于：
// Proxy.newProxyInstance(classLoader, arrayOf(ApiService::class.java)) { proxy, method, args ->
//     解析注解 → 构建 OkHttp Request → 执行请求 → 转换响应
// }
```

### 2. 核心组件

| 组件 | 职责 |
|------|------|
| `CallAdapter` | 将 `Call<T>` 适配为其他类型（suspend、Flow、RxJava） |
| `Converter` | 请求/响应体的序列化/反序列化（Gson、Moshi、kotlinx.serialization） |
| `CallFactory` | 发起 HTTP 请求（默认 OkHttpClient） |

```kotlin
val retrofit = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .client(okHttpClient)
    .addConverterFactory(GsonConverterFactory.create())  // JSON 转换
    .build()
```

### 3. 面试常问：Retrofit 如何支持 suspend 函数？

```kotlin
// Retrofit 2.6+ 原生支持 suspend
// 内部通过 KotlinExtensions 检测 suspend 函数
// 提取最后一个 Continuation 参数
// 使用 suspendCancellableCoroutine 桥接回调

// 简化版原理：
suspend fun <T> Call<T>.await(): T = suspendCancellableCoroutine { continuation ->
    enqueue(object : Callback<T> {
        override fun onResponse(call: Call<T>, response: Response<T>) {
            if (response.isSuccessful) {
                continuation.resume(response.body()!!)
            } else {
                continuation.resumeWithException(HttpException(response))
            }
        }
        override fun onFailure(call: Call<T>, t: Throwable) {
            continuation.resumeWithException(t)
        }
    })
    continuation.invokeOnCancellation { cancel() }
}
```

## 三、Glide 图片加载原理

> **面试常考**：Glide 三级缓存

### 1. 缓存机制

```
加载流程：
ActiveResources（活跃资源，弱引用）
    ↓ miss
MemoryCache（LruCache 内存缓存）
    ↓ miss
DiskCache（磁盘缓存）
    ├── ResourceCache（处理后的图片）
    └── DataCache（原始数据）
    ↓ miss
网络请求 / 本地文件
```

| 级别 | 存储位置 | 内容 | 特点 |
|------|---------|------|------|
| 活跃资源 | 内存（弱引用Map） | 正在使用的图片 | 使用中不会被回收 |
| 内存缓存 | 内存（LruCache） | 最近使用的图片 | LRU 淘汰策略 |
| 磁盘-资源 | 磁盘 | 变换后的图片 | 按尺寸、变换缓存 |
| 磁盘-数据 | 磁盘 | 原始数据 | 压缩原图 |

### 2. 生命周期管理

```kotlin
// Glide 通过添加隐藏 Fragment 监听生命周期
Glide.with(activity)    // 添加 RequestManagerFragment
Glide.with(fragment)
Glide.with(view)        // 获取 view 所在的 Activity/Fragment

// 生命周期绑定：
// onStart → 恢复请求
// onStop → 暂停请求
// onDestroy → 清除请求、释放资源
```

### 3. Glide 高级用法

```kotlin
Glide.with(context)
    .load(url)
    .placeholder(R.drawable.placeholder)     // 占位图
    .error(R.drawable.error)                 // 错误图
    .transition(DrawableTransitionOptions.withCrossFade()) // 淡入动画
    .override(300, 300)                      // 指定尺寸
    .transform(CircleCrop())                 // 圆形裁剪
    .diskCacheStrategy(DiskCacheStrategy.ALL) // 缓存策略
    .into(imageView)

// 预加载
Glide.with(context).load(url).preload()

// 清除缓存
Glide.get(context).clearMemory()            // 主线程
Glide.get(context).clearDiskCache()         // 子线程
```

## 四、网络安全基础

### HTTPS 握手流程

```
Client                          Server
  │──── ClientHello ────────→   │  （支持的TLS版本、加密套件、随机数）
  │                              │
  │←─── ServerHello ────────    │  （选择的版本、加密套件、随机数）
  │←─── Certificate ────────    │  （服务器证书）
  │←─── ServerHelloDone ───    │
  │                              │
  │──── ClientKeyExchange ──→   │  （用证书公钥加密的预主密钥）
  │──── ChangeCipherSpec ───→   │
  │──── Finished ───────────→   │  （用主密钥加密的验证数据）
  │                              │
  │←─── ChangeCipherSpec ───    │
  │←─── Finished ───────────    │
  │                              │
  │←─── 加密通信 ──────────→   │
```

### 证书锁定（Certificate Pinning）
