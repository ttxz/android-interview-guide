# 设计模式面试要点

## 一、六大设计原则（SOLID + 迪米特）

| 原则 | 含义 | Android 示例 |
|------|------|-------------|
| **S**ingle Responsibility | 一个类只负责一件事 | Activity 不做网络请求 |
| **O**pen/Closed | 对扩展开放，对修改关闭 | RecyclerView.Adapter |
| **L**iskov Substitution | 子类可替换父类 | Fragment 替换 |
| **I**nterface Segregation | 接口粒度要小 | View.OnClickListener |
| **D**ependency Inversion | 依赖抽象而非具体 | Repository 模式 |
| 迪米特法则 | 最少知识原则 | ViewModel 不持有 View |

### 1. 观察者模式

**Android 中的应用**：LiveData、Flow、RxJava、BroadcastReceiver、OnClickListener

```kotlin
// 核心思想：一对多依赖关系
interface Observer<T> {
    fun onChanged(value: T)
}

class Observable<T> {
    private val observers = mutableListOf<Observer<T>>()
    private var value: T? = null

    fun observe(observer: Observer<T>) { observers.add(observer) }
    fun removeObserver(observer: Observer<T>) { observers.remove(observer) }
    fun setValue(newValue: T) {
        value = newValue
        observers.forEach { it.onChanged(newValue) }
    }
}
```

### 2. 责任链模式

**Android 中的应用**：事件分发、OkHttp 拦截器链

```kotlin
abstract class Handler {
    var next: Handler? = null
    fun handle(request: Request): Response {
        return if (canHandle(request)) process(request)
        else next?.handle(request) ?: throw NoHandlerException()
    }
    abstract fun canHandle(request: Request): Boolean
    abstract fun process(request: Request): Response
}
```

## 四、面试中的设计题

### 实现一个简单的图片加载器

```kotlin
class ImageLoader private constructor(context: Context) {
    private val memoryCache = LruCache<String, Bitmap>(maxMemory / 8)
    private val diskCache = DiskLruCache.open(cacheDir, 1, 1, 50 * 1024 * 1024)
    private val executor = Executors.newFixedThreadPool(4)

    companion object {
        @Volatile private var INSTANCE: ImageLoader? = null
        fun with(context: Context) = INSTANCE ?: synchronized(this) {
            INSTANCE ?: ImageLoader(context.applicationContext).also { INSTANCE = it }
        }
    }

    fun load(url: String, imageView: ImageView) {
        // 1. 查内存缓存
        memoryCache.get(url)?.let { imageView.setImageBitmap(it); return }
        // 2. 查磁盘缓存
        // 3. 网络请求 → 解码 → 存缓存 → 主线程显示
        imageView.tag = url // 防止图片错位
        executor.execute {
            val bitmap = downloadAndDecode(url)
            memoryCache.put(url, bitmap)
            Handler(Looper.getMainLooper()).post {
                if (imageView.tag == url) imageView.setImageBitmap(bitmap)
            }
        }
    }
}

## 六、高频面试设计模式详解

### 装饰者模式（Decorator）

**Android 典型案例**：`ContextWrapper` 包装 `Context`

```kotlin
// 装饰者模式核心：包装对象，增强功能，不改原有接口
abstract class ContextWrapper(base: Context) : Context() {
    // 持有真实 Context 的引用
    private var mBase: Context? = base

    // 大多数方法委托给 mBase：这就是装饰的体现
    override fun getSystemService(name: String): Any? =
        mBase?.getSystemService(name)

    override fun startActivity(intent: Intent) =
        mBase?.startActivity(intent)
}

// Activity → ContextThemeWrapper → ContextWrapper → Context
// 每一层都在上一层基础上"包装"增加能力
```

### 工厂方法模式（Factory Method）

**Android 典型案例**：`LayoutInflater`、`Fragment.newInstance()`

```kotlin
// Fragment 工厂方法：把参数封装到 Bundle，避免直接 new Fragment(args)
class UserDetailFragment : Fragment() {
    companion object {
        // ✅ 标准工厂方法
        fun newInstance(userId: Long, userName: String): UserDetailFragment {
            return UserDetailFragment().apply {
                arguments = bundleOf(
                    "userId" to userId,
                    "userName" to userName
                )
            }
        }
    }
    // onCreateView 中从 arguments 读取参数
}

// 使用
val fragment = UserDetailFragment.newInstance(userId = 123L, userName = "张三")
```

> **为什么不用构造函数传参？**
> Fragment 被系统重建时会调用**无参构造函数**，通过构造传的参数会丢失。`arguments` Bundle 会由系统自动保存和恢复。

### OkHttp 拦截器链（责任链模式实战）

```kotlin
// 自定义拦截器
val loggingInterceptor = Interceptor { chain ->
    val request = chain.request()
    println("→ ${request.method} ${request.url}")
    val response = chain.proceed(request)  // 传递给下一个拦截器
    println("← ${response.code}")
    response
}

val authInterceptor = Interceptor { chain ->
    val original = chain.request()
    val request = original.newBuilder()
        .header("Authorization", "Bearer ${tokenProvider.getToken()}")
        .build()
    chain.proceed(request)
}

// OkHttp 内部拦截器链顺序：
// 自定义拦截器 → RetryAndFollowUpInterceptor → BridgeInterceptor
// → CacheInterceptor → ConnectInterceptor → CallServerInterceptor
val client = OkHttpClient.Builder()
    .addInterceptor(authInterceptor)   // 应用拦截器（最先执行）
    .addNetworkInterceptor(loggingInterceptor)  // 网络拦截器（连接后执行）
    .build()
```
