# Kotlin 基础语法与类型系统

## 一、基本类型

### 1. 数值类型

| 类型 | 位宽 | 范围 |
|------|------|------|
| `Byte` | 8 | -128 ~ 127 |
| `Short` | 16 | -32768 ~ 32767 |
| `Int` | 32 | -2^31 ~ 2^31-1 |
| `Long` | 64 | -2^63 ~ 2^63-1 |
| `Float` | 32 | IEEE 754 单精度 |
| `Double` | 64 | IEEE 754 双精度 |

### 2. 字符串模板

```kotlin
val name = "Kotlin"
val greeting = "Hello, $name!"                    // 简单引用
val length = "Name length: ${name.length}"        // 表达式
val multiLine = """
    |第一行
    |第二行
""".trimMargin()                                   // 原始字符串
```

### 3. 可空类型与空安全

> **面试高频考点**：Kotlin 如何解决 Java 中的 NullPointerException？

```kotlin
var nullable: String? = null       // 可空类型
var nonNull: String = "Hello"      // 非空类型

// 安全调用操作符
val length = nullable?.length      // 返回 null 而非抛异常

// Elvis 操作符
val len = nullable?.length ?: 0    // 空时给默认值

// 非空断言（危险，慎用！）
val len2 = nullable!!.length      // 空时抛 KotlinNullPointerException

// 安全类型转换
val num: Int? = value as? Int     // 转换失败返回 null
```

**底层实现原理**：

- 编译器在编译时通过类型检查强制区分可空与非空
- 非空类型在字节码中通过 `@NotNull` 注解标记
- `?.` 编译为 `if (obj != null)` 的条件判断

## 二、类与对象

### 1. 类的声明与构造函数

```kotlin
// 主构造函数 + init 块
class Person(val name: String, var age: Int) {
    init {
        require(age > 0) { "Age must be positive" }
    }

    // 次构造函数必须委托给主构造
    constructor(name: String) : this(name, 0)
}

// 数据类（自动生成 equals/hashCode/toString/copy/componentN）
data class User(val id: Long, val name: String, val email: String)

// 密封类（受限的类层次结构）
sealed class Result<out T> {
    data class Success<T>(val data: T) : Result<T>()
    data class Error(val exception: Throwable) : Result<Nothing>()
    object Loading : Result<Nothing>()
}
```

### 2. 面试常问：data class vs 普通 class

| 特性 | `data class` | 普通 `class` |
|------|-------------|-------------|
| `equals()` / `hashCode()` | 基于属性自动生成 | 基于引用（Object默认） |
| `toString()` | 包含所有属性 | 类名@hashCode |
| `copy()` | 自动生成 | 无 |
| `componentN()` | 自动生成（解构） | 无 |
| 继承 | 默认 `final`，不可被继承 | 可被继承 |

### 3. 对象声明与伴生对象

```kotlin
// 单例（线程安全，懒加载）
object DatabaseManager {
    fun getConnection(): Connection { /* ... */ }
}

// 伴生对象（类似 Java 的 static）
class MyClass {
    companion object {
        const val TAG = "MyClass"         // 编译时常量
        @JvmStatic                        // 生成真正的 static 方法
        fun create(): MyClass = MyClass()
    }
}

// 对象表达式（匿名内部类）
val listener = object : View.OnClickListener {
    override fun onClick(v: View?) { /* ... */ }
}
```

## 三、属性与委托

### 1. 属性的 getter/setter

```kotlin
class Rectangle(val width: Int, val height: Int) {
    // 计算属性（无 backing field）
    val area: Int
        get() = width * height

    // 带 backing field 的自定义 setter
    var label: String = ""
        set(value) {
            field = value.trim()  // field 关键字引用 backing field
        }
}
```

### 2. 属性委托

> **面试高频考点**：`by lazy` 和 `lateinit` 的区别？

```kotlin
// by lazy：线程安全的延迟初始化，仅用于 val
val heavyObject: HeavyObject by lazy {
    println("初始化只执行一次")
    HeavyObject()
}

// lateinit：用于 var，适合依赖注入等场景
lateinit var adapter: RecyclerView.Adapter<*>
// 检查是否已初始化
if (::adapter.isInitialized) { /* ... */ }

// 自定义委托
class PreferenceDelegate<T>(
    private val key: String,
    private val default: T
) : ReadWriteProperty<Any?, T> {
    override fun getValue(thisRef: Any?, property: KProperty<*>): T { /* 从 SP 读取 */ }
    override fun setValue(thisRef: Any?, property: KProperty<*>, value: T) { /* 写入 SP */ }
}

// 使用
var userName: String by PreferenceDelegate("user_name", "")
```

| 特性 | `by lazy` | `lateinit` |
|------|-----------|------------|
| 适用类型 | `val`（不可变） | `var`（可变） |
| 原始类型 | 支持 | **不支持** |
| 可空类型 | 支持 | **不支持** |
| 线程安全 | 默认 `SYNCHRONIZED` | 不保证 |
| 初始化检查 | 自动（首次访问时） | `::prop.isInitialized` |

## 四、扩展函数与扩展属性

```kotlin
// 扩展函数
fun String.isValidEmail(): Boolean {
    return Regex("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$").matches(this)
}

// 可空接收者扩展
fun String?.orEmpty(): String = this ?: ""

// 扩展属性
val String.lastChar: Char
    get() = this[length - 1]
```

**底层实现原理**：

- 扩展函数编译为 **静态方法**，第一个参数为接收者
- **不是**真正添加成员，**不能**访问私有成员
- **不具备多态性**：调用由编译时类型决定，而非运行时类型

```kotlin
open class Base
class Derived : Base()

fun Base.foo() = "Base"
fun Derived.foo() = "Derived"

val b: Base = Derived()
println(b.foo())  // 输出 "Base"（编译时类型是 Base）
```

## 五、接口与抽象类

### 面试常问：接口 vs 抽象类

| 特性 | 接口 (`interface`) | 抽象类 (`abstract class`) |
|------|-------------------|-------------------------|
| 构造函数 | 无 | 有 |
| 状态（字段） | 无 backing field | 可以有 |
| 多继承 | 支持 | 不支持 |
| 默认实现 | 支持 | 支持 |

```kotlin
interface Clickable {
    fun click()                              // 抽象方法
    fun showOff() = println("I'm clickable") // 默认实现
}

interface Focusable {
    fun showOff() = println("I'm focusable")
}

// 冲突解决
class Button : Clickable, Focusable {
    override fun click() { /* ... */ }
    override fun showOff() {
        super<Clickable>.showOff()  // 显式指定调用哪个
    }
}
```

## 六、枚举与密封类

### 面试常问：密封类 vs 枚举

```kotlin
// 枚举：固定常量集合
enum class Direction { NORTH, SOUTH, EAST, WEST }

// 密封类：受限的类层次结构，每个子类可携带不同数据
sealed class NetworkResult<out T> {
    data class Success<T>(val data: T) : NetworkResult<T>()
    data class Error(val code: Int, val message: String) : NetworkResult<Nothing>()
    object Loading : NetworkResult<Nothing>()
}

// 配合 when 使用，编译器检查是否穷尽
fun handle(result: NetworkResult<String>) = when (result) {
    is NetworkResult.Success -> println(result.data)
    is NetworkResult.Error -> println("${result.code}: ${result.message}")
    NetworkResult.Loading -> println("Loading...")
    // 不需要 else 分支，编译器知道已穷尽
}
```

| 对比 | `enum class` | `sealed class` |
|------|-------------|----------------|
| 实例数量 | 固定 | 可多个（data class子类） |
| 携带数据 | 所有实例结构相同 | 每个子类可不同 |
| 适用场景 | 简单状态枚举 | 复杂状态建模 |

## 七、作用域函数

> **面试高频考点**：`let`、`run`、`with`、`apply`、`also` 的区别？

| 函数 | 对象引用 | 返回值 | 典型使用场景 |
|------|---------|--------|-------------|
| `let` | `it` | Lambda 结果 | 空安全调用、局部变量转换 |
| `run` | `this` | Lambda 结果 | 对象配置并计算结果 |
| `with` | `this` | Lambda 结果 | 对已有对象调用多个方法 |
| `apply` | `this` | 对象本身 | 对象配置（Builder模式） |
| `also` | `it` | 对象本身 | 附加操作（日志、验证） |

```kotlin
// let：空安全 + 转换
val length = nullableString?.let { it.length } ?: 0

// apply：对象初始化
val textView = TextView(context).apply {
    text = "Hello"
    textSize = 16f
    setTextColor(Color.BLACK)
}

// also：链式附加操作
fun getUser() = repository.findUser()
    .also { log("Found user: $it") }
    .also { analytics.trackUser(it) }

// run：配置 + 计算
val result = service.run {
    port = 8080
    host = "localhost"
    start()  // 返回值
}
```
