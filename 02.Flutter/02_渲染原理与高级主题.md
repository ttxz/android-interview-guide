# Flutter 渲染原理与高级主题

## 一、Flutter 渲染引擎

### 1. 架构层次

```
Dart Framework（纯 Dart）
├── Material / Cupertino     → UI 组件库
├── Widgets                   → Widget 体系
├── Rendering                 → 布局和绘制
├── Animation / Gestures      → 动画和手势
└── Foundation                → 基础工具类

Flutter Engine（C++）
├── Skia                      → 2D 渲染引擎
├── Dart Runtime              → Dart VM
├── Text Layout (libtxt)      → 文字排版
└── Platform Channels          → 平台通信

Embedder（平台相关）
├── Android (Java/Kotlin)
├── iOS (Objective-C/Swift)
└── Web / Desktop
```

### 2. 渲染流水线

> **面试考点**：Flutter 如何渲染一帧？

```
VSync 信号 →
  1. Build Phase（Widget → Element → RenderObject）
  2. Layout Phase（布局计算，仅对标记 dirty 的节点）
  3. Paint Phase（绘制命令生成 Layer Tree）
  4. Composite Phase（合成图层）
  5. Rasterize Phase（Skia 光栅化 → GPU 渲染）
```

**关键点**：

- Flutter **自绘引擎**：不使用原生 View，直接通过 Skia 在 Canvas 上绘制
- 60fps / 120fps：每帧预算 16.6ms / 8.3ms
- 在独立线程上执行光栅化，避免阻塞 UI 线程

### 3. 三棵树详解

```dart
// Widget Tree：不可变的配置描述
// Element Tree：Widget 的实例，管理生命周期和子树更新
// RenderObject Tree：负责实际的布局（layout）和绘制（paint）

// 重建效率：
// Widget 重建 → Element 比较（canUpdate?）
//   → 相同类型 + 相同 key → 复用 Element，更新 RenderObject
//   → 不同 → 卸载旧 Element，创建新的

// canUpdate 判断
static bool canUpdate(Widget oldWidget, Widget newWidget) {
    return oldWidget.runtimeType == newWidget.runtimeType
        && oldWidget.key == newWidget.key;
}
```

### 2. 显式动画（Explicit）

```dart
class PulseAnimation extends StatefulWidget {
  @override
  State<PulseAnimation> createState() => _PulseAnimationState();
}

class _PulseAnimationState extends State<PulseAnimation>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _animation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: Duration(seconds: 1),
      vsync: this,  // TickerProvider，跟随屏幕刷新
    );
    _animation = Tween<double>(begin: 1.0, end: 1.5)
        .animate(CurvedAnimation(
            parent: _controller,
            curve: Curves.elasticOut,
        ));
    _controller.repeat(reverse: true);
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedBuilder(
      animation: _animation,
      builder: (context, child) {
        return Transform.scale(scale: _animation.value, child: child);
      },
      child: Icon(Icons.favorite, size: 48, color: Colors.red),
    );
  }

  @override
  void dispose() {
    _controller.dispose();  // 必须释放！
    super.dispose();
  }
}
```

## 三、Flutter 与原生混合开发

### 1. 混合栈管理

```dart
// FlutterEngineGroup：多引擎方案
// 共享 GPU 上下文和 Dart Isolate 减少内存开销

// 单引擎方案（FlutterFragment / FlutterActivity）
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val flutterFragment = FlutterFragment
            .withCachedEngine("my_engine")
            .build<FlutterFragment>()

        supportFragmentManager.commit {
            replace(R.id.container, flutterFragment)
        }
    }
}
```

### 2. 预热引擎

```kotlin
// Application 中预热 Flutter 引擎
class MyApp : Application() {
    lateinit var flutterEngine: FlutterEngine

    override fun onCreate() {
        super.onCreate()
        flutterEngine = FlutterEngine(this)
        flutterEngine.dartExecutor.executeDartEntrypoint(
            DartExecutor.DartEntrypoint.createDefault()
        )
        FlutterEngineCache.getInstance().put("my_engine", flutterEngine)
    }
}
```

### 3. 混合开发方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| FlutterActivity/Fragment | 官方支持 | 每次打开都要初始化 |
| 预热引擎（缓存） | 秒开 | 内存占用增加 |
| FlutterEngineGroup | 多引擎共享资源 | API 复杂 |
| FlutterBoost（闲鱼） | 完整的混合栈管理 | 第三方维护 |
